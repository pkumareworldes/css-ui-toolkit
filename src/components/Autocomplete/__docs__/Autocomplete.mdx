# Autocomplete

Autocomplete component with flexible props for single or multiple selection.

#### Example

<Canvas of={Autocomplete.Default} />

## Usage

```tsx
import { Autocomplete } from "sld-ui";

const options = [
  { label: "Option 1", value: 1 },
  { label: "Option 2", value: 2 },
  { label: "Option 3", value: 3 },
];

const Example = () => {
  return (
    <Autocomplete
      options={options}
      label="Choose an option"
      placeholder="Start typing..."
      helperText="Select an option from the dropdown"
      onChange={(event, value, reason) => console.log(value, reason)}
      multiple={false}
      size="medium"
    />
  );
};

export default Example;


#### Arguments
- options Array<{ label: string; value: string | number }> - An array of options for the autocomplete dropdown. Each option should have a label and value.
- label string - The label displayed above the input field.
- value AutocompleteOption | AutocompleteOption[] | null - The selected value(s) for the autocomplete. Can be a single option, multiple options, or null.
- onChange (event: SyntheticEvent, value: AutocompleteOption | AutocompleteOption[] | null, reason: AutocompleteChangeReason) => void - A callback triggered when the selected value changes. Receives the event, new value(s), and reason as arguments.
- placeholder string - Placeholder text displayed when the input field is empty.
- disabled boolean - Disables the autocomplete field, preventing user interaction. Default is false.
- multiple boolean - Enables selection of multiple options. Default is false.
- size string - Specifies the size of the input field. Possible values: "small", "medium". Default is "medium".
- helperText string - Additional text displayed below the input field for context or instructions.
- ...rest object - Additional props passed to the underlying MUI Autocomplete component.
